# Vault Structure

The vault is a plain markdown folder that serves as the agent's persistent knowledge base. It works with Obsidian, any markdown editor, or just a file browser.

## Folder Layout

```
<vault>/
├── BRAIN.md              # Instructions for how Claude uses the vault
├── Projects/             # One subfolder per registered project
│   ├── my-app/
│   │   ├── README.md     # Project status, architecture, decisions
│   │   └── CODEBASE.md   # Auto-generated by /index-codebase
│   └── api-server/
│       └── README.md
├── Sessions/             # Session handoff notes
│   ├── 2026-02-25-auth-refactor.md
│   └── 2026-02-24-bugfix-sprint.md
├── Reference/            # Stable reference docs
│   └── Dev-Commands.md
└── Planning/             # Goals, priorities, routines
    └── Goals.md
```

## What Goes Where

### Projects/
One subfolder per project you register in agent-os.config.json. Each should have at minimum a `README.md` with:
- What the project does (1-2 sentences)
- Current status
- Recent decisions
- Known blockers
- Next steps

The `/index-codebase` skill can auto-generate a `CODEBASE.md` with structure, dependencies, and conventions.

### Sessions/
Created by `/session-handoff` and `/session-cleanup`. These are context-continuity notes that let a fresh Claude session pick up where the last one left off.

Format: `YYYY-MM-DD-topic.md`

### Reference/
Stable information that doesn't change often. Dev commands, hardware specs, API references, etc.

### Planning/
Goals, quarterly priorities, daily routines. Checked by `/project-status` when generating reports.

## Starting Fresh

The `setup.js` script creates this structure automatically. You just need to:
1. Create a `README.md` in `Projects/<name>/` for each registered project
2. Write a brief description of what the project does and its current state
3. The system handles everything else (session notes, codebase indexing, status reports)
